================================================================================
PROJECT OVERVIEW
================================================================================

The Legal Contract Risk Analyzer is an end-to-end AI-powered system for analyzing
legal contracts using semantic search and machine learning. It identifies high-risk
clauses in PDF documents by:

1. Extracting text from PDF files
2. Converting clauses into semantic embeddings
3. Performing vector similarity search in Pinecone
4. Using ML-based risk classification
5. Providing chat-based Q&A about contract risks

Key Features:
- Vector Database Integration (Pinecone)
- Semantic Clause Similarity Search
- ML Risk Classification from 9,447 legal clauses
- PDF Processing and Analysis
- RESTful API with FastAPI
- Interactive Chat Interface
- Real-time Risk Analysis

api.py
  - Main FastAPI application
  - Endpoints:

    * POST /chat - Ask questions about analyzed contracts
    * GET /health - Health check
    * GET /docs - Interactive API documentation (Swagger)
  - Features:
    * CORS middleware for frontend communication
    * Multipart file upload handling
    * Error handling and logging
    * Session management

chat_orchestrator.py
  - **NEW: Direct RoBERTa QA Model Integration**
  - Functions:
    * _load_roberta_model() - Load local RoBERTa from models/roberta-base/
    * _run_roberta_qa() - Execute QA inference on question + context
    * answer_contract_question() - Main chat orchestration using RoBERTa
    * _get_top_relevant_clauses() - Rank clauses by keyword relevance
    * _is_summary_question() - Detect summary requests
    * _is_risk_distribution_question() - Detect risk analysis requests
  - **REMOVED**: Model server batch querying, fallback reasoning chains
  - **NEW**: Direct local model inference with torch
  - Process Flow:
    1. Load local RoBERTa model from models/roberta-base/
    2. Get top CHAT_CONTEXT_CLAUSES relevant clauses
    3. For each clause, run RoBERTa QA directly
    4. Return best answer by confidence score
  - Features:
    * GPU support (cuda if available, fallback to CPU)
    * Tensor-based inference
    * Direct model loading from local path
    * No external model service required
    * Efficient singleton model caching

chat_session_store.py
  - SQLite-based session management
  - Features:
    * create_session() - Start new chat session
    * add_message() - Store user/assistant messages
    * get_messages() - Retrieve conversation history
    * get_session() - Fetch session metadata
    * touch_session() - Update activity timestamp
    * cleanup_expired_sessions() - Remove old sessions
    * is_db_ready() - Check database status
  - Session TTL (Time-To-Live) configurable
  - Automatic cleanup of old sessions

model_server.py
  - Standalone QA model service
  - Runs on port 9000
  - Features:
    * Keeps transformer model loaded in memory
    * Handles batch question-answer processing
    * RESTful endpoints for model inference
    * Fallback support in main API
  - Optional but recommended for performance

FULL PIPELINE WORKFLOW
================================================================================

1. INITIALIZATION PHASE
   ├─ setup.py validates Python environment
   ├─ Creates .env configuration file
   ├─ Installs all dependencies
   └─ Runs system tests

2. DATA INGESTION PHASE (One-time)
   ├─ ingest_pipeline.py reads legal_contract_clauses.csv
   ├─ For each clause (9,447 total):
   │  ├─ embedder.py generates 768-dim semantic embedding
   │  └─ retriever.py uploads to Pinecone with metadata
   └─ Pinecone index ready for similarity search

3. RUNTIME SERVICES
   ├─ Backend API (api.py, port 8000)
   │  ├─ POST /analyze - Contract analysis
   │  ├─ POST /chat - Q&A about contract
   │  └─ GET /docs - API documentation
   │
   ├─ Optional Model Server (model_server.py, port 9000)
   │  └─ QA inference service (for performance)
   │
   └─ Frontend (React, port 5173)
      ├─ FileUpload component
      ├─ ResultsList component
      └─ ChatPanel component

4. CONTRACT ANALYSIS WORKFLOW

   User uploads PDF
        ↓
   pdf_extractor.py → Extract text + metadata
        ↓
   clause_segmenter.py → Identify clause boundaries
        ↓
   embedder.py → Generate semantic embeddings
        ↓
   risk_analyzer.py → Initial risk classification
        ↓
   retriever.py → Search Pinecone for similar clauses
        ↓
   For each high-risk clause found:
   ├─ Fetch clause details from Pinecone
   ├─ llm_reasoner.py scores confidence
   └─ retrieval_pipeline/main.py aggregates results
        ↓
   Results stored in session (chat_session_store.py)
        ↓
   API returns JSON response to frontend
        ↓
   Frontend displays ResultsList with ClauseCards

5. CHAT Q&A WORKFLOW (RoBERTa-Based)

   User asks question about contract
        ↓
   chat_session_store.py → Fetch conversation context
        ↓
   chat_orchestrator.py → Load local RoBERTa model from models/roberta-base/
        ↓
   _get_top_relevant_clauses() → Rank clauses by keyword relevance
        ↓
   For each relevant clause:
   ├─ Build context (clause + explanation + similar clauses)
   ├─ Run RoBERTa QA inference on question + context
   └─ Track confidence score from model
        ↓
   Select best answer by highest confidence score
        ↓
   If confidence >= CHAT_MIN_CONFIDENCE:
   ├─ Return extracted answer span from RoBERTa
   └─ Attach clause citations
        ↓
   If confidence < threshold (fallback):
   ├─ Return summary text from top-ranked clause
   └─ Mark as lower confidence
        ↓
   chat_session_store.py → Store Q&A pair
        ↓
   API returns answer to frontend ChatPanel

================================================================================
TROUBLESHOOTING GUIDE
================================================================================

Issue: "PINECONE_API_KEY not found"
  Solution: Add key to .env file
    PINECONE_API_KEY=pcsk_xxxxxxxxxxxxxxxxxxxxx
  Run: python scripts/setup.py

Issue: "Index not found in Pinecone"
  Solution: Run ingestion pipeline
    python backend/scripts/ingest_pipeline.py
  Time: ~5 minutes for 9,447 clauses

Issue: Model server timeout
  Solution: Disable model server temporarily
    $env:MODEL_SERVER_ENABLED="false"
  Or increase timeout:
    $env:MODEL_SERVER_TIMEOUT=30

Issue: PDF upload fails
  Solution: Verify:
    1. File is valid PDF
    2. File size < 50MB
    3. Text is extractable (not scanned)

Issue: Chat session expires
  Solution: Configure persistence
    Edit .env:
    CHAT_SESSION_TTL_SEC=7200  (2 hours)

Issue: Low confidence in answers
  Solution: Adjust threshold
    Edit config.py:
    CHAT_MIN_CONFIDENCE=0.5  (lower = more answers)

Issue: Slow analysis
  Solution:
    1. Enable model server batch mode
    2. Increase batch size in .env
    3. Check Pinecone connection
    4. Monitor RAM usage
